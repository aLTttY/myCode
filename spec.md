# MewCode MCP 客户端 Spec

## 背景

MewCode 已有统一的工具描述、注册、权限判定、执行和 Agent 结果回灌流程，但目前只能使用进程内置工具，不能发现或调用外部 MCP Server 提供的工具。用户需要手工开发本地工具才能扩展 Agent 能力，也无法复用 MCP 生态中的现有 Server。

本阶段为 MewCode 增加 MCP 客户端能力。用户在用户级或项目级配置中声明 MCP Server 后，MewCode 在启动期间与各 Server 建立会话、发现工具，并把可用工具接入现有工具中心。Agent 看到的 MCP 工具与内置工具采用同一种工具调用体验。

## 目标

- 支持通过本地 stdio 子进程和远程 Streamable HTTP 两种传输连接 MCP Server。
- 按 MCP 使用 JSON-RPC 2.0 完成会话初始化、工具发现和工具调用，并正确关联并发请求与响应。
- 将发现到的远端工具接入 MewCode 现有工具中心和 Agent Loop。
- 支持用户级、项目级两层 MCP Server 配置合并及环境变量展开。
- 缓存并管理多个 Server 的会话，使单个 Server 故障不影响本地工具、其他 Server 或 MewCode 启动。

## 功能需求

- F1: MewCode 必须从用户级 `~/.mycode/config.yaml` 和项目级配置读取 `mcp_servers`。项目级配置沿用默认的 `./config.yaml` 或 `--config` 指定文件；项目级同名 Server 定义整体覆盖用户级定义，非同名 Server 均予以保留。
- F2: `mcp_servers` 必须是以 Server 名为 key 的 map。每个 Server 必须显式声明 `transport: stdio` 或 `transport: http`，并且只能使用该传输允许的配置字段。
- F3: stdio Server 必须支持 `command`、`args` 和 `env`。启动子进程时继承 MewCode 当前进程环境，再由配置的 `env` 覆盖同名变量，并通过标准输入输出交换协议消息。
- F4: HTTP Server 必须支持 `url` 和 `headers`，并通过 MCP Streamable HTTP 传输交换协议消息。
- F5: stdio 的 `command`、`args`、`env` 值以及 HTTP 的 `url`、`headers` 值必须支持完整值或字符串内部的 `${VAR}` 展开。环境变量未设置时配置加载失败；变量已设置为空字符串时按合法值展开为空。
- F6: 配置结构错误、必填字段缺失、字段类型错误、传输字段混用、Server 名非法或环境变量未设置时，MewCode 必须报告可理解的配置错误并终止启动，不得带着含义不确定的配置继续运行。
- F7: 对每个已配置 Server，MewCode 必须在启动期间依次完成会话初始化和工具列表获取；只有成功完成这两个阶段的 Server 才能提供工具。
- F8: 协议消息必须遵循 JSON-RPC 2.0。客户端发出的每个请求必须带唯一 id，并能在响应异步到达时按 id 关联到原请求；通知和无关响应不得被误配为请求结果。
- F9: 每个已发现工具必须以 `<server>__<tool>` 作为 Agent 可见名称，并保留远端提供的描述和输入参数约束，使 Agent 能按普通工具方式选择和调用它。
- F10: Server 配置名必须能形成所有受支持 Provider 可接受的工具名。远端工具名或组合后的名称不合法时，MewCode 必须跳过该工具并给出包含 Server 和原因的警告，不得静默改名。
- F11: MCP 工具名称与已注册工具冲突，或同一 Server 返回重复工具时，必须保留先注册的工具、跳过冲突项并警告；不得覆盖已有工具，且不得影响其他合法工具注册。
- F12: Agent 调用 MCP 工具时，MewCode 必须复用对应 Server 的既有会话，向远端发起工具调用，并把响应转换为现有工具结果后回灌 Agent Loop，Agent 无需感知传输或协议差异。
- F13: MCP 工具结果必须支持文本内容和 `structuredContent`。文本内容进入可读工具消息，结构化内容保持原有数据语义；MCP 的 `isError: true` 必须映射为失败结果。
- F14: 本阶段无法消费的图片、音频、嵌入资源或资源链接结果不得静默丢弃或伪装为成功，必须返回说明不支持内容类型的结构化失败。
- F15: 所有 MCP 工具必须按有副作用的受控工具处理，进入现有权限判定并维持串行执行语义；本阶段不得依据远端声明的只读属性自动放行。
- F16: MewCode 必须按 Server 缓存活动连接和会话状态，同一 Server 的工具发现与多次工具调用复用该会话，不为每次调用重新启动进程或建立独立会话。
- F17: MewCode 退出时必须关闭活动会话、HTTP 连接和 stdio 管道，并结束由自身启动的子进程，避免遗留进程或占用连接。
- F18: 单个 Server 在进程启动、初始化、工具发现或后续调用期间失败时，失败必须局限于该 Server 或该次调用。本地工具和其他 Server 继续可用；即使全部 MCP Server 失败，MewCode 也继续启动并输出可理解警告。
- F19: 本阶段不执行自动重连。某个 Server 会话失效后，后续使用该会话的工具调用返回结构化失败，不得自动创建新会话掩盖故障。

## 非功能需求

- N1: MCP 客户端不得因单个 Server 无响应而无限阻塞 MewCode 启动、工具调用或退出；协议操作必须具有明确的超时边界。
- N2: 多 Server 会话及请求配对必须相互隔离；一个 Server 的响应、异常、退出或关闭不得完成、破坏或取消另一 Server 的请求。
- N3: 同一会话存在多个在途请求时，响应顺序与请求顺序不一致也必须得到正确结果。
- N4: MCP 适配不应改变内置工具的名称、行为、权限语义或既有 Provider 工具格式。
- N5: 配置错误、连接错误、协议错误、工具发现错误和工具调用错误必须可区分，并包含足够的 Server 上下文供用户定位问题，但不得泄露 header、环境变量或其他敏感值。
- N6: Server 输出、工具文本和结构化结果必须遵守 MewCode 既有工具结果大小边界，避免无界内容进入对话上下文。
- N7: 自动化测试必须覆盖两种传输、配置合并与展开、异步响应配对、工具适配、权限接入、会话复用、失败隔离和生命周期清理。

## 不做的事

- 本阶段不实现 MCP Resources 的发现、读取、订阅或模板能力。
- 本阶段不实现 MCP Prompts 的发现或获取。
- 本阶段不实现 MCP Sampling、Roots、Elicitation 或其他非工具能力。
- 本阶段不实现 Server 健康检查、心跳、后台探活、自动重连或自动重试会话创建。
- 本阶段不根据 MCP 工具注解自动判定只读或免审批。
- 本阶段不原生向模型传递图片、音频、嵌入资源或资源链接类型的工具结果。
- 本阶段不提供运行时动态新增、删除或重载 Server 配置的能力。
- 本阶段不提供 MCP Server 管理界面、状态面板或独立诊断命令。
- 本阶段不支持 stdio 与 Streamable HTTP 之外的传输。
- 本阶段不实现 MCP Server 本身，也不代理 Server 对外提供 MewCode 的内置工具。

## 验收标准

- AC1: 用户级和项目级配置含不同 Server 时两者均加载；存在同名 Server 时只使用项目级完整定义，且 `--config` 指定文件可作为项目级来源。（覆盖 F1）
- AC2: 合法的 stdio 与 HTTP 配置可加载；缺少 `transport`、必填字段缺失、字段类型错误或混用两种传输字段时启动失败并指出对应 Server 和字段。（覆盖 F2、F3、F4、F6）
- AC3: `${VAR}` 在独占值和嵌入字符串中均正确展开；未设置变量导致配置错误，已设置为空的变量展开为空；错误信息不包含其他敏感配置值。（覆盖 F5、F6）
- AC4: stdio 子进程可以读取继承的父进程环境，配置 `env` 可以覆盖同名变量，并能通过标准输入输出完成 MCP 会话。（覆盖 F3）
- AC5: Streamable HTTP Server 可以接收展开后的 URL 和 headers，并完成 MCP 会话；日志与错误不泄露 header 值。（覆盖 F4、F5）
- AC6: 对 stdio 和 HTTP Server，MewCode 启动时都按协议完成初始化后获取工具列表；任一步骤失败的 Server 不注册工具。（覆盖 F7）
- AC7: 同一 Server 的多个 JSON-RPC 请求响应乱序返回时，每个响应仍交付给 id 对应的请求；通知、未知 id 和重复响应不会误配。（覆盖 F8）
- AC8: 远端工具以 `<server>__<tool>` 出现在 Agent 可用工具列表中，描述和输入参数约束与 Server 返回内容一致，并可被 Agent 正常调用。（覆盖 F9、F12）
- AC9: Server 名非法会导致配置错误；远端工具名或最终名称非法时只跳过该工具并警告，同一 Server 的其他合法工具仍注册。（覆盖 F10）
- AC10: MCP 工具与内置工具、其他 MCP 工具或同一 Server 的重复工具发生最终名称冲突时，先注册项保持不变，冲突项被跳过并警告，其他工具不受影响。（覆盖 F11）
- AC11: 同一 Server 的连续工具调用复用同一活动会话，不重复启动 stdio 子进程或重新建立 MCP 会话。（覆盖 F12、F16）
- AC12: MCP 文本结果和 `structuredContent` 被完整转换为现有工具结果；`isError: true` 产生失败结果并正常回灌 Agent Loop。（覆盖 F13）
- AC13: 返回图片、音频、嵌入资源或资源链接时，调用得到包含不支持类型说明的结构化失败，且不把该结果报告为成功。（覆盖 F14）
- AC14: MCP 工具进入现有受控工具权限流程并串行执行；远端 `readOnlyHint` 不会使工具免审批或进入只读并发路径。（覆盖 F15）
- AC15: MewCode 正常退出后，HTTP 连接、stdio 管道以及由其启动的 MCP 子进程均被关闭；不存在遗留的待处理请求。（覆盖 F17）
- AC16: 一个 Server 启动失败、初始化失败、工具发现失败或调用失败时，其他 Server 和内置工具仍可用；所有 Server 均失败时 MewCode 仍可使用内置工具启动，并为每个失败 Server 输出警告。（覆盖 F18）
- AC17: 活动 Server 在会话期间退出或连接失效后，后续工具调用返回带 Server 上下文的结构化失败，且没有自动重连或隐式重启 Server。（覆盖 F19）
- AC18: Server 不响应时，启动、调用和退出均在配置或系统规定的超时边界内结束，不会无限等待。（覆盖 N1）
- AC19: MCP 接入后，六个既有内置工具的名称、描述、执行路径和权限行为通过回归测试。（覆盖 N4）
- AC20: 超大文本或结构化结果按现有工具输出边界截断或拒绝，不会无界写入 Agent 上下文。（覆盖 N6）
- AC21: 自动化端到端场景同时启动一个可用 stdio Server、一个可用 Streamable HTTP Server 和一个故障 Server；两个可用 Server 的工具均被发现、接受权限判定并成功回灌结果，故障 Server 仅产生警告，MewCode 最终无遗留连接或子进程。（覆盖 F1-F19、N1-N7）
