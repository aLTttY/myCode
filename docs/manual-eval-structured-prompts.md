# Structured Prompt Manual Evaluation

## 读取后编辑

输入：要求修改一个已存在文件中的一处文字。

观察点：模型是否先读取目标文件或搜索相关代码，再调用编辑工具。

通过标准：编辑前能看到读文件或搜索动作；编辑工具只在确认当前内容后使用。

## Plan Mode 只读

输入：`/plan 检查项目并给出修改方案`。

观察点：Provider 请求中的工具集合只包含 `read_file`、`find_files`、`search_code`，动态指令包含 Plan Mode 只读约束。

通过标准：写文件、改文件和执行命令工具没有开放；模型只输出计划。

## 专用工具优先

输入：要求分别找文件、读文件、搜索代码。

观察点：模型是否优先选择 `find_files`、`read_file`、`search_code`，而不是用 shell 命令替代。

通过标准：每类操作使用对应专用工具；只有专用工具不足时才使用命令。

## 环境变化缓存稳定

输入：用不同 cwd 或日期构造两次提示。

观察点：`stable_system_prompt` 是否保持一致，环境动态消息是否变化。

通过标准：稳定提示文本完全一致；`<mewcode_environment>` 中的 cwd 或日期反映新环境。

## 多轮动态注入

输入：构造三轮工具循环，重复间隔为 3。

观察点：第 1 轮和第 3 轮动态模式指令为完整规则，第 2 轮为精简提醒。

通过标准：`DynamicInstruction.full` 分别为 `True`、`False`、`True`，且每轮都以系统级动态消息发送。
