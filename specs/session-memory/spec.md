# MyCode 会话与长期记忆 Spec

## 背景

MyCode 当前能够在单个进程内维护多轮消息、卸载大型工具结果并压缩过长上下文，但进程退出后不会恢复对话。项目约定和用户偏好也没有稳定的跨会话入口，导致每次启动都需要重新说明背景，任务中断后难以平滑继续。

本阶段增加三层互补机制：手写项目指令提供明确且高优先级的长期约束；JSONL 会话存档保存可恢复的工作历史；自动笔记从自然完成的 Agent 轮次中沉淀用户偏好、纠正反馈、项目知识和参考资料。新请求发出前，MyCode 已加载适用的指令和记忆索引，并默认恢复当前项目最近的有效会话。

## 目标

- 在启动时恢复当前项目最近 30 天内的有效会话，同时允许用户显式开启新会话。
- 以安全、确定的优先级加载用户级和项目级手写指令，并支持受限的文件引用。
- 使用追加式 JSONL 保存完整会话历史，使单行损坏不会导致整个会话不可恢复。
- 在恢复时修复不完整的工具调用序列、控制上下文预算，并提醒模型关注长时间中断带来的状态变化。
- 在 Agent 自然完成一轮后异步沉淀长期笔记，并在后续请求前注入有界的记忆索引。
- 严格分离用户级与项目级记忆，避免项目特例污染其他项目。

## 功能需求

### 项目指令

- F1: MyCode 启动时必须尝试加载三份约定的手写 Markdown 指令：`<项目>/.mycode/MYCODE.md`、`<项目>/MYCODE.md` 和 `~/.mycode/MYCODE.md`。文件不存在时按空内容处理，不要求用户预先创建。
- F2: 指令必须按 `.mycode/` 项目指令、项目根指令、用户级指令的顺序进入模型上下文。该顺序同时表达从高到低的冲突优先级，模型必须被明确告知优先遵循靠前内容。
- F3: 指令文件必须支持 `@include` 引用其他文本文件，并在引用位置展开内容。项目级指令及其后代引用只能读取当前项目真实路径范围内的文件；用户级指令及其后代引用只能读取 `~/.mycode/` 真实路径范围内的文件。
- F4: `@include` 最多递归 5 层。加载过程必须记录已经访问的真实路径，同一文件只展开一次；环路、超深引用、越界路径、符号链接越界、缺失文件和不可读文件必须被安全跳过并产生不包含文件正文的可理解告警，不得阻止其余合法指令加载。
- F5: 每次新进程启动必须重新读取指令；运行期间不要求监视或热重载文件变化。

### 会话存档与选择

- F6: 项目会话必须存放在 `<项目>/.mycode/sessions/`，每个会话使用一个独立 JSONL 文件。会话 ID 必须采用 `YYYYMMDD-HHMMSS-xxxx`，其中随机后缀用于避免同一秒创建时碰撞。
- F7: 会话记录必须只以追加方式写入，每行是一个可独立解析的完整 JSON 记录。系统不得为会话另建 meta 文件，也不得通过重写整个会话文件来更新标题、消息数或最后活动时间。
- F8: 存档必须包含恢复合法对话序列所需的用户消息、助手消息、助手工具调用和工具结果，并保留它们的关联标识与顺序。每条完整记录完成后应及时刷新，使进程异常中断最多影响正在写入的最后一行。
- F9: 会话 ID、标题、消息数、创建时间和最后活动时间等派生信息必须在需要时直接扫描 JSONL 计算。标题取第一条有效用户消息的可展示摘要；没有有效用户消息的文件不得成为自动恢复候选。
- F10: 默认启动必须自动恢复当前项目 30 天内最后活动时间最新的有效会话。启动参数 `--new` 必须跳过自动恢复并创建新会话；交互命令 `/new` 必须完成当前轮次的收尾后切换到新会话。
- F11: 执行 `/new` 或正常退出前，当前会话中已经完成的消息必须保持持久化；会话切换不得删除旧会话，后续启动仍可按规则恢复它。
- F12: 每次启动必须扫描并删除最后一条有效记录早于当前时间 30 天以上的会话文件。清理失败必须告警但不得阻止启动；空文件或完全损坏且无法确定时间的文件必须跳过自动恢复，并可按文件修改时间参与过期清理。

### 会话恢复

- F13: 恢复扫描遇到无法解析、结构非法或类型不受支持的 JSONL 行时必须跳过该坏行并继续检查后续记录，同时报告坏行数量而不输出其原始内容。
- F14: 恢复后的消息必须形成合法的模型对话序列。若助手工具调用缺少对应结果、工具结果找不到对应调用，或坏行造成关联断裂，必须从首个无法组成完整工具调用单元的位置截断后续恢复历史，不得把孤立工具记录发送给模型。
- F15: 恢复历史在发送下一次普通模型请求前必须接受现有上下文预算检查。超限时最多自动执行一次压缩；若仍超预算，必须完整保留会话但阻止普通模型请求，并提示用户再次执行 `/compact` 或通过 `/new` 开始新会话。
- F16: 恢复时若最后一条有效记录距当前时间超过 24 小时，下一次请求必须加入一条系统级时间跨度提醒，包含可理解的间隔信息并提示外部状态可能变化。该提醒不得伪装成用户消息，也不得反复追加进原始会话历史。
- F17: 自动恢复、`--new`、`/new`、坏行跳过、历史截断、压缩结果和时间跨度提醒必须通过简洁状态信息对用户可见，但不得输出存档中的消息正文或工具结果。

### 自动笔记与索引

- F18: 自动笔记必须分为四类：用户偏好、纠正反馈、项目知识和参考资料。每条笔记使用一个独立的、带 frontmatter 的 Markdown 文件，正文包含足以理解和复用该记忆的简洁内容。
- F19: 项目级笔记必须存放在 `<项目>/.mycode/memory/`，用户级笔记必须存放在 `~/.mycode/memory/`；两个作用域各自维护一份 Markdown 索引。项目目录下的 sessions 和 memory 目录必须默认被 Git 忽略，手写 `MYCODE.md` 不受此忽略规则影响。
- F20: 每次 Agent Loop 以“模型最终回复且没有工具调用”的自然完成状态停止后，MyCode 必须异步调用当前会话使用的 LLM 分析本轮，并决定是否新增、更新、合并或忽略笔记。达到最大迭代、取消、流错误、工具解析错误、未知工具阈值或上下文超限等非自然停止不得触发自动笔记更新。
- F21: 自动笔记的去重和合并判断必须交给 LLM，LLM 在判断时可读取目标作用域的现有索引和必要笔记。重复事实不得仅因措辞不同而持续产生新文件。
- F22: LLM 必须按适用范围选择存储作用域：跨项目偏好和通用纠正写入用户级；仅适用于当前仓库的偏好、纠正、项目知识和参考资料写入项目级。无法可靠判断时必须默认项目级。
- F23: 自动笔记更新不得延迟或改变当前轮的最终回复。更新成功后应对后续请求可见；更新失败只产生安全告警，不得破坏会话或已有笔记。
- F24: 执行 `/new` 或正常退出时，如果自动笔记更新仍在进行，系统必须在有限时间内等待完成；超时后允许放弃本次更新并告警，不得无限阻塞会话切换或退出。
- F25: 用户级和项目级索引都必须同时限制在 200 行以内且不超过 25KB。接近或超过任一上限时，必须让 LLM 重建精简索引，合并重复项、缩短摘要并优先保留高价值和近期条目；原始笔记不得因索引精简被删除。若 LLM 结果仍超限，必须使用确定性规则裁剪到两个上限内。
- F26: 处理每个普通用户请求前，MyCode 必须把当前项目级索引和用户级索引作为系统上下文注入，使 Agent 无需先调用文件工具即可利用已索引记忆。项目级索引必须排在用户级索引之前；索引缺失时按空内容处理。
- F27: LLM 必须被明确禁止把 API Key、访问令牌、密码、私钥或其他凭据写入自动笔记。任何笔记和索引落盘前还必须经过确定性的敏感内容检查；疑似包含秘密时整条更新必须被拒绝并产生不含原文的告警。

## 非功能需求

- N1: 指令、会话和记忆的所有项目级文件访问必须限制在当前项目真实路径内；用户级记忆与指令访问必须限制在 `~/.mycode/` 真实路径内，不得通过 `..` 或符号链接越界。
- N2: 会话追加、笔记写入和索引替换必须避免留下被当作有效内容的半写文件。单条失败不得破坏此前已持久化的数据。
- N3: 恢复和索引处理必须具有确定的顺序；同一组文件在内容未变化时应产生一致的候选选择、消息顺序和兜底裁剪结果。
- N4: 启动必须能容忍单个指令文件、会话文件、笔记文件或索引文件损坏。局部错误应隔离并告警，其余合法数据继续可用。
- N5: 自动笔记请求必须复用当前 Provider、模型和凭据，但不得携带可执行工具；模型输出中的任何工具调用都不得执行。
- N6: 自动笔记异步任务不得无界并发。连续自然完成的轮次必须有明确顺序，且同一索引的更新不得互相覆盖。
- N7: 状态、错误和清理日志不得打印原始会话正文、工具结果、完整笔记正文、凭据或其他敏感内容。
- N8: 新能力不得改变现有 Provider 消息合法性、Agent Loop、权限系统、MCP 工具、Plan/Do 模式、流式输出和上下文压缩的既有行为，除非本规格明确要求恢复或注入额外上下文。
- N9: 自动化测试必须覆盖三层指令优先级与引用安全、JSONL 追加和损坏恢复、工具调用截断、会话选择与过期清理、恢复压缩与时间提醒、四类笔记与作用域、异步生命周期、去重、索引边界和秘密拦截。

## 不做的事

- 本阶段不引入向量数据库、embedding、RAG、语义检索或按当前问题动态召回笔记。
- 本阶段不实现团队记忆服务、跨设备同步、云端会话同步或通过 Git 分发自动记忆。
- 本阶段不提供会话或记忆的图形管理界面。
- 本阶段不提供会话重命名、删除、搜索、导出或任意历史会话选择器；默认只恢复最近有效会话，用户可新建会话。
- 本阶段不热重载 `MYCODE.md`、自动笔记或索引；它们在规定的启动或请求边界重新读取。
- 本阶段不自动删除因索引压缩而暂时未出现在索引中的原始笔记。
- 本阶段不保证机器断电时正在写入的最后一条 JSONL 记录或正在进行的异步笔记更新一定完成。
- 本阶段不从会话内容中构建全文检索库、统计数据库或单独的元数据文件。

## 验收标准

- AC1: 三份 `MYCODE.md` 同时存在且包含冲突指令时，模型上下文按 `.mycode/` 项目级、项目根、用户级顺序呈现，并明确高优先级优先；缺失任一文件不影响其他文件加载。（覆盖 F1、F2、F5）
- AC2: 合法 `@include` 在原位置展开；重复引用只出现一次。构造直接环路、间接环路、第 6 层引用、`..` 越界和符号链接越界时，违规引用被跳过并产生安全告警，其他合法内容仍加载。（覆盖 F3、F4、N1）
- AC3: 新会话生成符合 `YYYYMMDD-HHMMSS-xxxx` 的唯一 ID，并在 `.mycode/sessions/` 形成逐行可独立解析的 JSONL；不存在配套 meta 文件。（覆盖 F6、F7）
- AC4: 完成包含工具调用的一轮后，扫描 JSONL 可还原用户、助手、工具调用和工具结果的原始顺序与关联；标题、消息数和时间信息均可仅通过该文件计算。（覆盖 F8、F9）
- AC5: 当前项目存在多个有效会话时，普通启动恢复 30 天内最后活动时间最新者；`--new` 不恢复旧会话，`/new` 持久化当前已完成内容后切换到新 ID，旧文件保持可恢复。（覆盖 F10、F11）
- AC6: 启动时自动删除超过 30 天的会话，保留未过期会话；单个文件删除失败时给出告警且 CLI 仍能启动。（覆盖 F12）
- AC7: 在 JSONL 中插入坏行后，恢复会跳过坏行并继续读取合法记录；坏行造成工具结果缺失或加入孤立工具结果时，恢复历史从首个不完整工具单元起截断，发送给三类 Provider 的序列仍合法。（覆盖 F13、F14、N4）
- AC8: 恢复历史超出上下文预算时只自动压缩一次；压缩成功后可继续请求，仍超限时 Provider 未收到普通请求、原存档未丢失且用户看到 `/compact` 与 `/new` 提示。（覆盖 F15）
- AC9: 恢复距最后活动超过 24 小时的会话时，下一次请求包含一次系统级时间提醒；24 小时以内不包含，连续请求不会把同一提醒反复写入会话或伪装成用户消息。（覆盖 F16、F17）
- AC10: 一轮自然完成后，最终回复立即展示，后台笔记请求使用当前模型且不执行工具；非自然停止不触发笔记请求。失败时已有会话和笔记保持不变。（覆盖 F20、F23、N5）
- AC11: 分别给出用户偏好、纠正反馈、项目知识和参考资料示例时，生成的 Markdown 笔记具有 frontmatter 和对应分类；跨项目内容进入用户级，项目特定内容进入项目级，不确定内容进入项目级。（覆盖 F18、F19、F22）
- AC12: 对语义重复但措辞不同的多轮输入，LLM 更新已有笔记或忽略新增，不持续创建重复文件；并发或连续更新不会丢失先完成的索引修改。（覆盖 F21、N6）
- AC13: `/new` 和正常退出遇到正在运行的笔记任务时会有限等待；任务及时完成则索引可见，超时则安全退出并告警，不出现无限等待。（覆盖 F24）
- AC14: 构造超过 200 行或 25KB 的索引后，系统先精简重建并在必要时确定性裁剪，最终同时满足两个上限，原始笔记文件数量不因该过程减少。（覆盖 F25）
- AC15: 普通请求发出前同时包含项目级与用户级记忆索引，项目级在前；模型无需文件工具即可引用索引中的信息，缺失索引不影响请求。（覆盖 F26）
- AC16: 自动笔记候选包含测试 API Key、令牌、密码和私钥片段时，候选及关联索引更新均不落盘，终端只显示不含秘密正文的拒绝告警。（覆盖 F27、N7）
- AC17: `.mycode/sessions/` 和 `.mycode/memory/` 默认被 Git 忽略，而项目根及 `.mycode/` 下的 `MYCODE.md` 可正常跟踪。（覆盖 F19）
- AC18: OpenAI、Anthropic 和 DeepSeek 在全新会话、恢复会话、带时间提醒、带两级记忆索引和压缩后的请求中均保持合法消息顺序；现有工具、权限、MCP、Plan/Do、流式输出和上下文测试通过回归。（覆盖 N8）
- AC19: 端到端场景中，用户完成带工具调用的任务并自然退出，随后间隔超过 24 小时重新启动：最近会话自动恢复，坏行被跳过，不完整工具尾部被截断，时间提醒和两级索引在下一请求前注入，新完成轮次异步更新正确作用域的笔记，且整个过程不依赖 meta 文件。（覆盖 F1-F27、N1-N9）
